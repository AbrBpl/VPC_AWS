# VPC with transit gateway

Configuration in this directory creates a VPC resource with private, public and transit gateway subnets (one per availability zone). Private subnets have internet access via a NAT gateway while transit gateway subnets don't.

The topology is composed by three availability zones (with one NAT Gateway on each).

Transit gateway subnets are configured as per [AWS transit gateway design best practices](https://docs.aws.amazon.com/vpc/latest/tgw/tgw-best-design-practices.html).

> Note: Transit gateway configuration not shown as it is outside the scope of this module.

## Usage

To run this example you need to execute:

```bash
$ terraform init
$ terraform plan
$ terraform apply
```

Note that this example may create resources which can cost money (AWS Elastic IP, for example). Run `terraform destroy` when you don't need these resources.
